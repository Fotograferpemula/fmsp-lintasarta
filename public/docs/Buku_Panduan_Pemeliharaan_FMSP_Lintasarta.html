2.3 Middleware & Library Keamanan

Setiap API endpoint dilindungi oleh chain middleware berikut:

  1. Rate Limiter: Membatasi jumlah request per IP per window (login: 5 req/min, API: 60 req/min)
  2. JWT Verifier (auth-middleware): Memvalidasi token dari header Authorization, cek expired, cek user aktif
  3. RBAC Enforcer (rbac-middleware): Memeriksa apakah role user memiliki permission untuk resource + action
  4. Input Validator (validators/Zod): Memvalidasi dan sanitize body request — strip HTML tags, escape special chars
  5. Audit Logger: Mencatat operasi ke tabel AuditLog

3.2 Docker Compose (Development)

Untuk menjalankan development environment lokal:

docker compose up -d        # Start PostgreSQL container
npm install                 # Install dependencies
npx prisma db push          # Sync schema ke database
npx prisma db seed          # Seed data awal (SuperAdmin, AppSettings)
npm run dev                 # Start Next.js dev server (port 4100)

3.3 Integrasi Google SSO

Konfigurasi Google OAuth Client ID:

  1. Buka Google Cloud Console → APIs & Services → Credentials
  2. Buat OAuth 2.0 Client ID (Application type: Web application)
  3. Set Authorized JavaScript Origins: http://localhost:4100 dan https://fmsp-lintasarta.fly.dev
  4. Set Authorized Redirect URIs: http://localhost:4100 dan https://fmsp-lintasarta.fly.dev
  5. Salin Client ID ke GOOGLE_CLIENT_ID di .env
ℹ️ Catatan: Google SSO memanfaatkan Google 2-Step Verification yang sudah aktif di akun Google Workspace pengguna, sehingga MFA otomatis terpenuhi.

3.4 Konfigurasi AI Copilot (Ollama)

  • OLLAMA_BASE_URL: Alamat server Ollama (default: http://127.0.0.1:11434)
  • OLLAMA_MODEL: Model yang digunakan (default: qwen2.5:1.5b)
  • Status on/off dapat dikendalikan secara dinamis melalui AppSetting key ai_bot_enabled
  • Fallback otomatis ke keyword search jika Ollama tidak merespons dalam 1.2 detik

4.3 Health Check & Auto-scaling

Fly.io melakukan health check HTTP GET ke path / setiap 15 detik dengan timeout 10 detik dan grace period 20 detik saat startup. Jika health check gagal, instance di-restart otomatis. Concurrency limit: soft=80, hard=100 koneksi per instance.

6. Prosedur Backup & Restore

6.1 Manual Backup (pg_dump)

Pencadangan database dilakukan dengan utilitas pg_dump:

Backup dari Database Lokal (Docker)

pg_dump -h localhost -U fmsp_user -d fmsp_db \
  -F c -b -v \
  -f backup/fmsp_backup_$(date +%Y%m%d_%H%M%S).dump

Backup dari Fly.io (Production)

# 1. Port-forward database Fly.io ke lokal
fly proxy 5432 -a fmsp-lintasarta-db

# 2. Jalankan pg_dump ke localhost
pg_dump -h localhost -p 5432 -U fmsp_user -d fmsp_db \
  -F c -b -v \
  -f backup/fmsp_prod_$(date +%Y%m%d_%H%M%S).dump

6.2 Manual Restore (pg_restore)

pg_restore -v -h <host> -U <user> -d <db> \
  --clean --no-owner \
  /path/to/backup_file.dump

Flag --clean menghapus tabel lama sebelum restore. Flag --no-owner mencegah error ownership pada environment berbeda.

6.3 Backup Otomatis (Cron)

Direkomendasikan memasang cron job pada server database:

# Cron setiap hari pukul 01:00 WIB
0 1 * * * /app/scripts/backup-db.sh >> /var/log/fmsp_backup.log 2>&1

# Retensi: simpan 30 backup terakhir, hapus yang lebih lama
find /backup/ -name "fmsp_*.dump" -mtime +30 -delete

7. Keamanan Sistem

7.1 Autentikasi & JWT

  • Password di-hash menggunakan bcrypt dengan cost factor tinggi
  • JWT ditandatangani dengan HS256 menggunakan JWT_SECRET
  • Token berisi payload: { userId, email, role, region }
  • Expired time diatur pada saat penandatanganan
  • Google SSO memverifikasi token via Google OAuth API endpoint

7.2 RBAC & Permission Matrix

Didefinisikan di src/lib/rbac.ts — 6 role dengan matriks permission per resource per action. Enforced oleh middleware src/lib/rbac-middleware.ts pada setiap API route.

7.3 Rate Limiting

Implementasi in-memory rate limiter per IP address:

  • Login endpoint: 5 request per 1 menit
  • SSO endpoint: 10 request per 1 menit
  • Upload endpoint: 10 request per 1 menit
  • General API: 60 request per 1 menit

7.4 XSS & Input Validation

Semua input API divalidasi oleh Zod 4 schemas (src/lib/validators.ts). Fitur proteksi:

  • String sanitization: strip tag <script>, escape HTML entities
  • Email validation via regex ketat
  • Enum validation untuk field status, role, tipe dokumen
  • Number range validation (positif, max limit)
  • File upload: MIME type whitelist, magic bytes verification, max 10MB

7.5 CORS & HTTP Headers

  • CORS: Restricted ke domain yang terdaftar saja
  • Error responses di-sanitize — tidak mengekspos stack trace atau path server
  • API error handler terpusat (src/lib/api-error.ts) mengembalikan format JSON konsisten

10. Troubleshooting

MasalahPenyebabSolusi
Aplikasi tidak bisa start JWT_SECRET masih default Set JWT_SECRET unik: openssl rand -hex 64
Error "Cannot connect to database" DATABASE_URL salah / PostgreSQL down Verifikasi koneksi: psql $DATABASE_URL
Google SSO error "Invalid client_id" GOOGLE_CLIENT_ID tidak valid Cek di Google Cloud Console → Credentials
Email notifikasi tidak terkirim Konfigurasi SMTP salah Test via Admin → Pengaturan SMTP → Kirim Test Email
User terkunci (locked) 5x gagal login Tunggu 15 menit atau reset via Admin → Manajemen User
AI Copilot offline Server Ollama tidak berjalan ollama serve lalu ollama pull qwen2.5:1.5b
Migration error di Fly.io Schema conflict Backup → fly ssh console → npx prisma db push --force-reset
Upload gagal File > 10MB atau tipe tidak diizinkan Max 10MB. Tipe: PNG, JPG, PDF, DOCX
Cron tidak jalan CRON_SECRET tidak diset Set CRON_SECRET di .env dan di external cron caller
Push notification tidak muncul VAPID keys tidak valid Regenerate: npx web-push generate-vapid-keys
